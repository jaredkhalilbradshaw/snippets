# Reusable proofs of concept

These are intentionally dependency-free, one-file extractions of the most reusable ideas in Do to Done.
Each file owns its sample data, implementation, and executable checks.

- They encode product behavior, not project-specific rendering.
- Their invariants are already exercised in the main app and its tests.
- Each can transfer to features and flows of new or existing software.
- The files mostly use plain HTML or modern JavaScript with no package dependencies.
